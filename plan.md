我希望编写一个 neta 经典的 MBTI 性格测试的 NXTI （Nix-TI）测试，测试用户是哪种类型的 nix 用户。

题目量大概在 40 题左右。结果类型使用幽默风趣搞怪的标题，配合一本正经的文本描述。

不需要 4 个维度，而是将维度分数隐藏起来，来匹配最贴切的人格。下面给出一些人格结果类型的例子，你需要将这些结果扩充到大约 20 个：

- NERD: 书呆子
- PLMaster: PL 达人
- FPLover: FP-Lover 纯函数式编程狂热爱好者，题目中测试是不是喜欢不可变性
- ARCH: 隔壁 arch linux 用户
- XYN: 小药娘（添加一个玩笑题目，问“我在服用雌二醇”）
- PERF: 性能狂，使用最高性能编译所有包
- IDK: i dont know 我也不知道为什么我就用 nix 了
- PRAG: 实用主义者
- DEVOPS：Dev ops 狂人
- ENV: 纯粹用 nix 管理环境
- LBRD: 老板让的，老板/公司说要用 nix，自己不想用

## Plan: NXTI 人格测试站点

从零搭建一个 Vite + React + TypeScript + Tailwind 的单页测试站点，核心是约 40 题的隐藏维度加权匹配机制，输出 Top 3 搞怪人格结果，并将答题进度持久化到 localStorage。推荐把“MBTI 式四维”彻底藏在实现里，前端只展示题目、进度、结果卡片和一本正经的文案。

**Steps**

1. Phase 1 - 项目基线：初始化 pnpm + Vite + React + TypeScript + Tailwind；补齐基础样式、页面壳、移动端适配和最小目录结构。这一步阻塞后续所有内容。

2. Phase 2 - 测试数据模型：定义问题、选项、隐藏权重、结果类型、结果描述的数据结构；约 40 题按主题分桶（配置哲学、不可变性、打包/缓存、开发体验、部署/运维、社区文化、公司约束）。这一步依赖步骤 1。

3. Phase 3 - 人格体系：把现有结果扩充到约 20 个，统一为“短代号 + 荒诞标题 + 正经描述 + 特征标签 + 命中逻辑偏好”；加入 Top 3 排名规则与平分处理。可与步骤 2 并行设计，但落地代码依赖步骤 1。

需要设计一个可扩展的人格体系数据结构，方便后续增加新的人格类型和调整匹配逻辑。

4. Phase 4 - 测试流程 UI：实现开始页、答题页、进度条、上一题/下一题、可恢复状态提示、提交结果页；隐藏底层维度分数，只输出人格解释。依赖步骤 2 和步骤 3。

5. Phase 5 - 本地持久化：将当前题号、答案、时间戳、结果缓存进 localStorage；实现页面刷新恢复、完成后重测、版本变更时清理旧数据。依赖步骤 4。

6. Phase 6 - 结果体验：实现主结果 + Top 3 候选卡片、幽默副标题、一本正经长文案、推荐语和“为什么像你”摘要；确保不暴露内部维度分数。依赖步骤 3、4、5。

7. Phase 7 - 验证与收尾：跑构建和类型检查，手测刷新恢复、断点续答、结果稳定性、移动端布局和本地存储回放。依赖前面全部步骤。

**Verification**

1. 运行依赖安装、开发启动、生产构建与类型检查，确认空仓库初始化后的工程可正常运行。

2. 手动验证：答到一半刷新页面，状态应恢复到原题号和已选答案。

3. 手动验证：完成测试后重新进入页面，应展示结果或允许一键重测，行为按设计一致。

4. 手动验证：多组极端答案应得到不同人格，且只显示 Top 3，不显示任何内部维度数值。

5. 手动验证：桌面端和移动端都能完整完成 40 题流程，无布局溢出。

**Decisions**
- 技术栈使用 Vite + React + TypeScript + Tailwind。
- 结果页显示 Top 3 候选人格，而不是只给单一结果。
- 题目和结果采用“搞怪标题 + 严肃描述”的反差风格。
- 不实现敏感属性推断；若要保留相似荒诞气质，改写为非敏感人格，例如“Rice 仙人”“Dotfiles 炼金术士”“Flake 传教士”等社区梗角色。
- 当前范围默认是纯前端单页应用，不包含后端、登录、联网存档或分享海报生成功能。

**Further Considerations**

1. 评分建议使用“多人格加权命中”而不是传统四维折算：每题选项直接给若干人格原型加减权重，这样更容易做出幽默、非对称、隐藏维度的结果分布。

2. localStorage 建议带 schemaVersion，避免后续你改题库后旧答案结构导致恢复失败。
